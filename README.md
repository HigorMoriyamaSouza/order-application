# Order Application

Backend service for managing transactional order data using a RESTful API.

## Overview

Order Application is a backend service designed to manage orders and related data through a structured REST API. The project demonstrates core backend engineering concepts including relational database modeling, API design, and SQL-based data management.

The system allows clients to create, update, retrieve, and delete order records while maintaining structured relationships between entities such as orders and products.

This project was built as a practical demonstration of backend development skills and database interaction in a typical transactional service environment.

## Features

* RESTful API for order management
* CRUD operations for orders
* Relational database modeling
* SQL queries for data manipulation and retrieval
* Structured backend architecture
* Integration with MySQL database

## Technologies Used

* **Node.js** – Runtime environment
* **Express.js** – Web framework for building the API
* **MySQL** – Relational database for persistent data storage
* **PhpMyAdmin** – Database administration and query management

## API Capabilities

The API supports common operations for order management:

* Create new orders
* Retrieve order records
* Update existing orders
* Delete orders

These operations are implemented through RESTful endpoints and interact with the underlying MySQL database using SQL queries.

## Database

The project uses a relational database model to store order information and related entities. The schema was designed to ensure data consistency and support efficient querying for transactional data.

## Learning Goals

This project was created to practice and demonstrate:

* Backend service development
* REST API design principles
* SQL query development
* Relational data modeling
* Structured application architecture

## Project Structure

The application follows a modular structure typical of backend services:

* Route definitions for API endpoints
* Controllers for request handling
* Database interaction layer
* Configuration for server and database connection
